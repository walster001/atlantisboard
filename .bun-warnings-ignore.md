# Bun Build Warnings - Expected Behavior

## Tailwind CSS v4 Warnings

When building the client, you may see warnings like:
```
warn: invalid @ rule encountered: '@theme'
warn: invalid @ rule encountered: '@tailwind'
warn: invalid @ rule encountered: '@plugin'
warn: Invalid selector. Unsupported pseudo-class or pseudo-element 'details-content'
warn: Invalid selector. Unsupported pseudo-class or pseudo-element 'picker'
```

**These warnings are harmless and can be safely ignored.**

### Why They Occur

1. **Tailwind v4 Directives**: Tailwind CSS v4 uses `@import 'tailwindcss'`, `@plugin`, and `@theme` directives. Bun's CSS bundler doesn't recognize these as standard CSS at-rules, but they are processed correctly by Tailwind's PostCSS plugin during the build.

2. **DaisyUI Pseudo-Selectors**: DaisyUI 5 uses modern CSS pseudo-classes like `:details-content`, `:picker`, and `:picker-icon` that Bun's CSS bundler doesn't recognize. These are valid CSS features that browsers support, but Bun's parser is more conservative.

### Browser Compatibility

These pseudo-selectors are supported in all modern browsers:
- `:details-content` - Part of the CSS Details/Summary specification
- `:picker` - Part of the CSS Pseudo-Elements Module Level 4
- `:picker-icon` - Part of the CSS Pseudo-Elements Module Level 4

**No action needed** - these warnings do not affect functionality. The CSS works perfectly in browsers.
